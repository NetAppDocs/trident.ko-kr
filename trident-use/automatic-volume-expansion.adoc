= 자동 볼륨 확장
:allow-uri-read: 


자동 볼륨 확장을 사용하면 Trident에서 프로비저닝한 영구 볼륨(PV)이 사용된 용량이 정의된 임계값에 도달할 때 자동으로 확장됩니다.

이 기능은 *Autogrow Policies(AGP)*를 사용하여 구현됩니다. Autogrow Policy는 다음을 정의합니다:

* 확장을 트리거하는 사용률 임계값
* 볼륨이 증가하는 양
* 볼륨이 도달할 수 있는 최대 크기


자동 볼륨 확장 기능은 운영 오버헤드를 줄이고 볼륨 용량 부족으로 인한 애플리케이션 중단을 방지하는 데 도움이 됩니다.



== 요구 사항

* Trident 26.02 이상
* RBAC 권한을 생성하여 `TridentAutogrowPolicy` 사용자 정의 리소스
* StorageClasses 다음이 포함되어야 합니다. `allowVolumeExpansion: true`
* 지원되는 ONTAP 프로토콜:
+
** NFS 를 참조하십시오
** iSCSI
** FCP
** NVMe





NOTE: ONTAP 9.16.1 이전 버전의 ONTAP NVMe raw 블록 볼륨은 자동 확장을 지원하지 않습니다.


NOTE: SAN 볼륨의 경우,  `growthAmount`이(가) 50MiB 이하이면 Trident는 크기 조정 전에 값을 51MiB로 증가시킵니다. 단, 결과 크기가  `maxSize`을(를) 초과하지 않는 경우에 한합니다.


NOTE: 브라운필드 환경에서는 볼륨 게시 마이그레이션 동작으로 인해 특정 기존 볼륨에 대해 자동 확장이 작동하지 않을 수 있습니다.



== 자동 증가 정책 생성

Autogrow 정책은  `TridentAutogrowPolicy` 유형의 Kubernetes 사용자 지정 리소스입니다.

자동 증가 정책은 볼륨이 언제, 어떻게 확장되는지를 정의합니다.



=== 필드 정의

[cols="1,3,1"]
|===
| 필드에 입력합니다 | 설명 | 필수 요소입니다 


| `metadata.name` | 고유 정책 식별자. | 예 


| `usedThreshold` | 확장을 트리거하는 사용된 용량의 백분율입니다. | 예 


| `growthAmount` | 임계값에 도달했을 때 증가할 양입니다. 백분율(예: `"10%"`) 또는 고정 크기(예: `"5Gi"`)로 지정할 수 있습니다. 지정하지 않으면 기본값은 `10%`입니다. | 아니요 


| `maxSize` | 최대 볼륨 크기 제한. 지정하지 않으면 볼륨은 강제 상한 없이 증가할 수 있습니다. | 아니요 
|===


=== 예: 표준 자동 증가 정책

다음 예시는 사용률이 80%에 도달하면 확장이 시작되고, 10%씩 증가하며, 증가량을 500Gi로 제한하는 정책을 생성합니다.

[source, yaml]
----
apiVersion: trident.netapp.io/v1
kind: TridentAutogrowPolicy
metadata:
  name: standard-autogrow
spec:
  usedThreshold: "80%"
  growthAmount: "10%"
  maxSize: "500Gi"
----


=== 정책을 생성합니다

. YAML 정의를 파일에 저장합니다.
. 정책을 적용합니다.


[listing]
----
kubectl apply -f autogrow-policy.yaml
----
. 생성 확인:


[listing]
----
kubectl get tridentautogrowpolicy standard-autogrow
----
예상 출력:

[listing]
----
NAME                USED THRESHOLD   GROWTH AMOUNT   STATE
standard-autogrow   80%              10%             Success
----


== 구성 예



=== 운영 데이터베이스에 대한 보수적인 정책

용량 고갈을 방지하는 것이 중요한 운영 데이터베이스 워크로드의 경우 더 낮은 임계값과 더 큰 증가 증분 값을 사용하십시오.

[source, yaml]
----
apiVersion: trident.netapp.io/v1
kind: TridentAutogrowPolicy
metadata:
  name: production-db-policy
spec:
  usedThreshold: "75%"
  growthAmount: "20%"
  maxSize: "5Ti"
----


=== 고정 증가량을 가진 로그 스토리지

예측 가능한 성장 단계를 선호하는 경우 고정 크기 증분 방식을 사용하십시오.

[source, yaml]
----
apiVersion: trident.netapp.io/v1
kind: TridentAutogrowPolicy
metadata:
  name: log-storage-policy
spec:
  usedThreshold: "90%"
  growthAmount: "10Gi"
  maxSize: "100Gi"
----


=== 기본값이 포함된 자동 증가 정책

 `usedThreshold`만 지정된 경우 `growthAmount`는  `10%`로 기본 설정되며  `maxSize`는 제한이 없습니다.

[source, yaml]
----
apiVersion: trident.netapp.io/v1
kind: TridentAutogrowPolicy
metadata:
  name: logs-policy
spec:
  usedThreshold: "85%"
----


=== 기본 growthAmount를 사용한 자동 증가 정책

기본 증가량을 사용하면서 최대 크기를 지정하십시오.

[source, yaml]
----
apiVersion: trident.netapp.io/v1
kind: TridentAutogrowPolicy
metadata:
  name: default-ga-policy
spec:
  usedThreshold: "70%"
  maxSize: "100Gi"
----


=== 기본 maxSize를 사용한 자동 증가 정책

무제한 증가를 허용하면서 사용자 지정 증가 비율을 지정합니다.

[source, yaml]
----
apiVersion: trident.netapp.io/v1
kind: TridentAutogrowPolicy
metadata:
  name: default-ms-policy
spec:
  usedThreshold: "80%"
  growthAmount: "150%"
----


=== 소수 백분율을 사용한 자동 증가 정책

소수점 단위의 백분율을 사용하면 임계값과 증가량을 세밀하게 제어할 수 있습니다.

[source, yaml]
----
apiVersion: trident.netapp.io/v1
kind: TridentAutogrowPolicy
metadata:
  name: fractional-policy
spec:
  usedThreshold: "80.28%"
  growthAmount: "10.65%"
  maxSize: "100Gi"
----


== 정책 상태

[cols="1,3"]
|===
| 상태 | 설명 


| 성공 | 정책이 검증되어 사용 준비가 완료되었습니다. 


| 실패했습니다 | 유효성 검사 오류가 감지되었습니다. 사양을 검토하고 수정하십시오. 


| 삭제 중 | 삭제 작업이 진행 중입니다. 볼륨이 정책을 참조하는 동안 최종화 도구가 삭제를 방지합니다. 
|===


== 자동 증가 정책 관리



=== 정책 나열

[listing]
----
kubectl get tridentautogrowpolicy
----


=== 정책 세부 정보 보기

[listing]
----
kubectl describe tridentautogrowpolicy production-db-policy
----


=== tridentctl 사용

[listing]
----
tridentctl get autogrowpolicy
tridentctl get autogrowpolicy production-db-policy -o yaml
----


=== 자동 증가 정책 업데이트


IMPORTANT: 변경 사항은 현재 해당 정책을 사용 중인 모든 볼륨에 적용됩니다.

[listing]
----
kubectl edit tridentautogrowpolicy production-db-policy
----
업데이트는 다음 증가 작업 중에 적용됩니다. 볼륨을 다시 시작할 필요가 없습니다.



=== 자동 증가 정책 삭제

자동 증가 정책에는 사용 중 삭제를 방지하기 위한 finalizer 보호 기능이 포함되어 있습니다.

[listing]
----
kubectl delete tridentautogrowpolicy production-db-policy
----
볼륨에서 정책을 참조하는 경우, 모든 참조가 제거될 때까지  `Deleting` 상태가 유지됩니다.

정책을 사용하여 볼륨을 식별하려면:

[listing]
----
tridentctl get autogrowpolicy production-db-policy -o yaml
----
볼륨에서 정책 제거:

[listing]
----
kubectl annotate pvc <pvc-name> \
  trident.netapp.io/autogrowPolicy="none" \
  --overwrite
----


== StorageClass에서 자동 증가 정책을 구성합니다

 `trident.netapp.io/autogrowPolicy` 주석을 사용하여 정책을 연결합니다.

[source, yaml]
----
apiVersion: storage.k8s.io/v1
kind: StorageClass
metadata:
  name: ontap-gold
  annotations:
    trident.netapp.io/autogrowPolicy: "production-db-policy"
provisioner: csi.trident.netapp.io
parameters:
  backendType: "ontap-san"
  fsType: "ext4"
allowVolumeExpansion: true
----
주석 확인:

[listing]
----
kubectl get storageclass ontap-gold \
  -o jsonpath='{.metadata.annotations.trident\.netapp\.io/autogrowPolicy}'
----


=== StorageClass에서 자동 증가 정책 업데이트

[listing]
----
kubectl annotate storageclass ontap-gold \
  trident.netapp.io/autogrowPolicy="standard-autogrow" \
  --overwrite
----


=== StorageClass에서 자동 증가 정책 제거

옵션 1: 주석 제거

[listing]
----
kubectl annotate storageclass ontap-gold \
  trident.netapp.io/autogrowPolicy-
----
옵션 2: 주석을 다음으로 설정합니다 `none`

[listing]
----
kubectl annotate storageclass ontap-gold \
  trident.netapp.io/autogrowPolicy="none" \
  --overwrite
----


== PVC에 Autogrow 정책 적용

PVC 수준에서 정책을 적용하여 StorageClass 동작을 재정의합니다.

[source, yaml]
----
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: database-pvc
  annotations:
    trident.netapp.io/autogrowPolicy: "production-db-policy"
spec:
  storageClassName: ontap-gold
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 500Gi
----
PVC에서 자동 증가 비활성화:

[listing]
----
kubectl annotate pvc database-pvc \
  trident.netapp.io/autogrowPolicy="none" \
  --overwrite
----

== 문제 해결

[cols="1,3,2"]
|===
| 문제 | 가능한 원인 | 해결 


|  `Failed` 상태의 정책 | 잘못된 사양 | 정책을 수정하고 다시 적용하십시오 


| 볼륨이 확장되지 않습니다 | `maxSize` 도달했습니다 | 증가 `maxSize` 


| 잦은 확장 | 임계값이 너무 낮음 | 증가 `usedThreshold` 


| PVC가 표시됩니다 `AutogrowPolicyNotFound` | 정책이 존재하지 않습니다 | 정책 이름 확인 


| PVC에 다음이 표시됩니다 `AutogrowPolicyNotUsable` |  `Failed` 상태의 정책 | 유효성 검사 오류 해결 


| TAGRI는 반복적으로 거부했습니다 | `maxSize` 초과했습니다 | 늘리 `maxSize`거나 조정하세요 `growthAmount` 
|===
